## Issues

### [config] Single confluenceParentPageId can't accommodate separate PRD and PPP locations
- **Step:** `/prd-to-ppp`
- **Observed:** PRDs and PPPs typically live under different parent pages, but `CLAUDE.local.json` only has one `confluenceParentPageId`.
- **Fix needed:** Split into two fields — `confluenceParentPageIdPRD` and `confluenceParentPageIdPPP` (or similar). Update `CLAUDE.local.json.template`, `setup-check` field table, `to-prd`, and `prd-to-ppp` skills accordingly.

### [workflow] prd-to-ppp duplicates milestone logic instead of delegating to /milestones
- **Step:** `/prd-to-ppp`
- **Observed:** `prd-to-ppp` independently asked about milestone refinement — duplicating what `/milestones` does.
- **Expected workflow:**
  1. `/to-prd` → PRD
  2. `/milestones` → refine milestones (standalone, re-runnable)
  3. `/prd-to-ppp` → create PPP using already-refined milestones from step 2
- **Fix needed:** `prd-to-ppp` should not do milestone refinement itself. It should either (a) invoke `/milestones` as a sub-step, or (b) assume milestones have already been refined and just consume them. `/milestones` stays independently re-runnable for add/delete/update as the project evolves.

### [prd-to-ppp] Asks for PPP parent page at runtime instead of reading from config
- **Step:** `/prd-to-ppp`
- **Observed:** At the end, Claude asked "Where should the PPP page live? Give me the parent Confluence page URL" — but this should be read from `CLAUDE.local.json` (`confluenceParentPageIdPPP`).
- **Fix needed:** `prd-to-ppp` skill must read parent page from config, not prompt the user.

### [prd-to-ppp] PPP created from scratch instead of copying the template
- **Step:** `/prd-to-ppp`
- **Observed:** Skill created a blank PPP page instead of copying the designated template page.
  - Template (Copy Me): https://newrelic.atlassian.net/wiki/spaces/VERTSOL/pages/4689068188/Project+Plan+Copy+Me
  - First (wrong) PPP created: https://newrelic.atlassian.net/wiki/spaces/VERTSOL/pages/5714509983/Project+Plan+AFA+Onboarding+UI+Enhancements
  - Correct PPP (after user manually triggered copy flow): https://newrelic.atlassian.net/wiki/spaces/VERTSOL/pages/edit-v2/5714280543
- **Expected:** Skill should always copy the template page and then populate it with content — never create from scratch.
- **Fix needed:**
  1. Add `confluencePPPTemplatePageId` to `CLAUDE.local.json.template` so the template page ID is configured once.
  2. `prd-to-ppp` skill must copy that template page first, then write content into it.

### [prd-to-ppp] Wrong expansion of "PPP"
- **Step:** `/prd-to-ppp`
- **Observed:** Claude expanded PPP as "Plan, Progress, Problems" page.
- **Expected:** PPP = "Project Planner Page" (internal term).
- **Fix needed:** All skills that reference PPP should spell it out as "Project Planner Page" at first use so Claude never guesses.

### [to-prd] PRD created under wrong parent — used requirements doc instead of confluenceParentPageId
- **Step:** `/to-prd`
- **Observed:** Claude searched for space ID dynamically, got `3714908381` (not from `CLAUDE.local.json`). Created PRD as child of the requirements doc shared in conversation, not under the configured `confluenceParentPageId`.
- **Created at:** https://newrelic.atlassian.net/wiki/spaces/VERTSOL/pages/5715492953/PRD+Onboarding+UI+Enhancements
- **Expected:** PRD should be created as a child of `confluenceParentPageId` from `CLAUDE.local.json`.
- **Root cause hypothesis:** Skill derives parent from conversation context (most recently mentioned page) instead of reading `CLAUDE.local.json`.
- **Fix needed:** `to-prd` SKILL.md must explicitly instruct Claude to read `CLAUDE.local.json` and use `confluenceParentPageId` as the parent — not infer it from conversation context.

### [setup-check] confluenceSpaceKey may be redundant given confluenceParentPageId
- **Step:** `/setup-check`
- **Observed:** `confluenceSpaceKey` and `confluenceParentPageId` appear to overlap — if you have the parent page ID, the space can be derived from it. User had to ask Claude to explain the difference.
- **Claude's explanation:**
  - `confluenceSpaceKey` — short identifier in URL (`/wiki/spaces/MYTEAM/...`). Used to scope where docs are published.
  - `confluenceParentPageId` — numeric ID of the parent page (`/pages/123456789/...`). All PRDs/PPPs nest under this.
- **Question to resolve:** Do any skills actually use `confluenceSpaceKey` independently, or is it always derivable from the parent page? If derivable, remove it to reduce setup friction.
- **Fix needed:** Audit skills for `confluenceSpaceKey` usage. If unused or redundant, remove from template and setup-check. If needed, add clear descriptions so user doesn't have to ask.



### [setup-check] CLAUDE.local.json template lacks field descriptions
- **Step:** `/setup-check`
- **Observed:** When prompted to fill in `CLAUDE.local.json`, the template only shows the keys with no explanation of what each key means or where to find the values.
- **Expected:** Each field should have an inline comment or the setup-check output should describe what each key is for (e.g., `confluenceParentPageId` — the numeric ID of the Confluence page under which PRDs/PPPs will be created).
- **Fix needed:** `.claude/templates/CLAUDE.local.json.template` and/or the `setup-check` SKILL.md field table should add a description column explaining each key.
